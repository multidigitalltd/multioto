מחבר את אתר הוורדפרס לפאנל התפעול של Multi Digital: נקודת קצה MCP מאובטחת
לאבחון ותיקון מרחוק — תמיד רק אחרי אישור צוות — ועדכון עצמי של התוסף.

== מאיפה מורידים את קובץ ה-ZIP ==

הדרך הקלה: בפאנל, במסך האתר, לוחצים "הורד תוסף (גרסה אחרונה)" — מקבלים את ה-ZIP.
(גם ב-repo: wordpress-plugin/releases/multioto-agent-<גרסה>.zip, ובנייה מחדש: wordpress-plugin/build.sh.)

== התקנה וחיבור ==

כל הקודים מוכנים בפאנל — רק מעתיקים אותם לתוסף. אין צורך להמציא מפתח או טוקן.

1. בפאנל: הורידו את התוסף ("הורד תוסף") והתקינו באתר ("תוספים → הוסף חדש → העלאת תוסף") והפעילו.
2. בפאנל, במסך האתר, לחצו "קודי חיבור לתוסף" — ייפתח חלון עם ארבעה ערכים מוכנים להעתקה:
   כתובת הפאנל, כתובת MCP, מפתח MCP, טוקן עדכון.
3. באתר, ב"הגדרות → Multi Digital Agent", הדביקו את אותם ארבעה ערכים ושמרו.
   (מפתח MCP חייב להיות זהה לזה שבפאנל — פשוט מעתיקים אותו.)
4. בפאנל: ודאו ש"חיבור פעיל" דלוק ולחצו "בדוק חיבור AI".

== אבטחה ==

* כל בקשה מהפאנל מאומתת במפתח ה-MCP (השוואת hash בזמן קבוע).
* התוסף חושף רשימה סגורה של פעולות בלבד — אין הרצת קוד/SQL/קבצים שרירותית.
* קריאת הגדרות מוגבלת לרשימת מפתחות בטוחה (ללא סודות/PII).
* שינויים (עדכון/הפעלה/כיבוי תוסף, ניקוי מטמון) מבוצעים רק לאחר אישור מנהל
  בפאנל — התוסף עצמו לא יוזם דבר.

== עדכון עצמי ==

התוסף בודק מול הפאנל אם יש גרסה חדשה ומתעדכן דרך מנגנון העדכונים הרגיל של
וורדפרס. אין צורך להתקין ידנית מחדש בכל אתר בכל גרסה.

== שרתים שמסירים את כותרת Authorization ==

חלק מהשרתים (PHP-CGI/FastCGI) מסירים את כותרת Authorization. אם "בדיקת החיבור"
נכשלת עם 401 למרות מפתח נכון, הוסיפו ל-.htaccess:

  RewriteEngine On
  RewriteCond %{HTTP:Authorization} ^(.*)
  RewriteRule .* - [E=HTTP_AUTHORIZATION:%1]
